## New surfaces, new verdicts

- **Heart save:** `ProductCard.tsx:170` aria-label flips between `'Save to Love'` and `'Saved to Love'`. The verb is "Save," the lane is "Love." That's load-bearing taxonomy from elsewhere in the app, not chirp. The fill state is rose; no toast, no "Item saved!" banner, no 🎉. This is the right amount of nothing. CollageView (`CollageView.tsx:190`) mirrors it. Pass.
- **Pair-with:** `ProductCard.tsx:317-324`. Label is `"Pair with…"` (ellipsis, sentence case, no enthusiasm). The aria-label is `"Pair with — what would go with ${product.title}?"` — a sentence, not a marketing pitch. The function (`:70-75`) sends `"what would go with this? (product: ${title}, id: ${id})"` to the agent. That's a *user-voice* message, not a *bot-voice* command — when this scrolls into the chat history it reads like I asked it, which is true. Polo Ralph, not Echo. Pass.
- **Empty-results card:** `MessageRenderer.tsx:84-98`. *"Nothing matched for `<query>`. Try fewer constraints, or paste an image of what you have in mind."* No apology, no "Oops!", no "I couldn't find...". Two concrete next moves, one of which is the moodboard upload — that's a deflection back into product surface area, not into more chat. I'd shave one word: "Try fewer constraints" → "Try fewer constraints." (it already has the period) — fine as-is. Pass with no notes.
- **Ethics chip grid:** `PreferencesCard.tsx:548-557`. The eight values are `Sustainable / Fair-trade / Organic / B-Corp / Women-owned / Small-batch / Vegan / Recycled`. That reads editorial — these are vocabulary words from the values-led shopping world, not marketing claims. The grid does NOT include `Eco-friendly`, `Conscious`, `Mindful`, `Clean`, `Ethical` (the empty-calorie words). The active state is emerald, not the commerce orange, which means a saved ethics filter doesn't visually compete with a Buy CTA. The helper text *"Tap to toggle · Save to commit"* (`:683`) is mechanical, not chirpy. Pass.
- **Made-in line:** `MerchantBlock.tsx:116-121`. `"Made in {country}"` with a `MapPin` glyph. Sits in the same `text-xs text-ink-600` rhythm as the shipping line above it. When absent, it joins the comma-list at `:75` (`"Merchant didn't publish country of origin"`). This is trust-building: the value is there if present, the absence is named if not. It is not a flag emoji, it is not "Proudly handcrafted in...". Pass. Strong move actually.
- **Trust-promise disclosure:** `InputBar.tsx:178-182` — *"Prices and availability come from Shopify merchants via the Catalog MCP. Ranking is preference-driven, not paid placement."* Two lines of `text-[11px] text-ink-400` under the input. **The first line says "via the Catalog MCP."** Nobody outside this codebase knows what MCP is. Strip it: *"Prices and availability come from Shopify merchants."* The second line is the load-bearing one and it lands — it stakes a specific anti-claim ("not paid placement") that's verifiable from the rules engine (`reasoning.ts` has no ad-rank code path). Not smug. Just-right when the MCP jargon is gone.
- **Quota copy:** `useConversation.tsx:307-310` overrides the BE's rate-limit string to *"Hitting traffic — try again in a moment. If this keeps happening, daily quota may be exhausted."* Two problems. (1) The word *"quota"* is a backend implementation detail leaking into user copy — civilians read "quota" and think "did I do something wrong?" or "is this a free tier I just used up?" (2) The whole sentence is the agent talking *about itself* in a sheepish voice — *"hitting traffic"* is what a friend says when they're late, not what a tool says when it's rate-limited. Re-write: *"The model is busy right now. Try again in a minute. If this keeps happening, today's request budget may be used up."* Even that's wordy. Honestly: *"Service busy. Try again in a minute."*
- **Share page / gist:** `app/s/[id]/page.tsx:46-51`, rendered in `SummaryHero.tsx:40-42` as `font-display text-3xl italic`. If I typed *"good kitchen knives, German or Japanese, around $200"* the hero is exactly that sentence in italic serif. Reads humble, not raw. The italic + serif treatment elevates a literal user query into something that looks deliberate — *that's* the editorial move. The opposite of a fortune-cookie summary. Pass and keep passing.

## Recommend-to-a-friend rating (0–10) — and what would push it to a 9+

**8 / 10.**

Round 1 I would have said 7 with two asterisks. The Sparkles is gone, the welcome message is rewritten, the ToolStatus fallback is bounded. Three of my five original tells are dead. The new surfaces (heart, Pair-with, empty card, ethics grid, Made-in, trust disclosure) all landed with the same restraint the chips have — no chirp, no marketing voice, real data behind every value. The share-page gist is still the user talking back to themself, which remains the single most trust-earning sentence in the app.

What would push it to a 9: kill the two surviving chatbot-voice surfaces. The header subtitle ("Conversational product discovery") is the masthead apologising for itself. The error strings ("Try again?" / "Hitting traffic" / "daily quota may be exhausted") are the agent narrating its own internal state in a tone borrowed from Slack support bots. Both are easy to fix and both are the kind of thing I'd quote when explaining the app to a friend — which means the friend would hear the apologetic tone before they ever ran a query.

What would push it to a 10: the prose between chips. I never got to actually run a query in this walk, but my round-1 worry stands — the system prompt forbids describing what the user can already see but doesn't forbid *evaluative adjectives* ("great pick," "perfect for your kitchen"). One more line in `prompts.ts` — *"Do not use evaluative adjectives. Describe the result set, not whether it's good."* — would close the last gap.
